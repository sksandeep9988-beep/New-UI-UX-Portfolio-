01 — Overview

What is This Project?

A real-world HMI (Human Machine Interface) UI designed during my tenure at Adroit Control Engineering — a company that builds industrial electrical control panels.

Industrial electrical panels used in factories and commercial buildings house multiple AC units. Traditionally, these were controlled by physical switches on the panel itself — operators had to manually check each unit. This project involved designing a touchscreen HMI UI that replaces physical controls with a single digital interface, enabling engineers to monitor and control all AC units from one screen.

02 — Industry Context

Understanding the Domain

HMI (Human Machine Interface) design for industrial control systems is one of the most specialized UI disciplines — where errors can cause equipment failure or safety hazards.

Adroit Control Engineering designs and manufactures industrial electrical panels used in factories, data centres, hospitals, and commercial complexes. These panels contain multiple AC units that regulate temperature in critical environments. Each unit needs to be individually controllable — both automatically (based on sensor data) and manually (by operator override). The UI had to run on a touchscreen embedded in the panel door, which means every element had to be large enough for glove-friendly touch, readable under industrial lighting, and fast to operate under pressure.

🏭
Industrial Environment

Panel mounted in factories — high-noise, low-lighting, glove operation conditions.

⚡
Mission Critical

AC failure in server rooms or hospitals is a serious operational risk. The UI must prevent errors.

👷
Non-Tech Users

Operators are electrical engineers or technicians — not software users. Clarity > aesthetics.

03 — Problem & Solution

What Problem Does This Solve?

Replacing physical switch panels with a digital HMI that gives complete control, status visibility, and automation in one screen.

⚠ Before — Physical Panel Problems

  • No single view of all AC unit statuses — operators had to check each unit separately
  • Physical switches gave no feedback — was the unit ON or just switched?
  • No automation — all switching was manual, requiring constant operator presence
  • No error alerts — faults went unnoticed until equipment damage occurred
  • No logs or history — impossible to audit what happened when
  • Factory reset required physical panel access and technical knowledge

✅ After — HMI UI Solution

  • Single dashboard shows all AC units, their ON/OFF state and mode at a glance
  • Clear visual feedback — green/red indicators, toggle states, real-time status
  • Auto mode — units respond to sensor data without operator intervention
  • System failure screen with acknowledgement — immediate fault visibility
  • Trends page — historical data visualization for auditing and diagnostics
  • Factory default reset via secure confirmation screen with login protection

07 — Key UX Decisions

Why These Design Choices?

🟢

Semantic Color — Green/Red Only for Status

In industrial UI, colors carry life-or-death meaning. Green = running safely. Red = fault or danger. Amber = operator attention needed. Blue = automated. These colors are never used decoratively — each use is a functional signal. An operator glancing at the panel for 2 seconds must instantly know the system state.

🔘

Large Toggle Switches — Glove-Friendly Design

Industrial operators often wear safety gloves. Standard mobile-style small toggle switches are impossible to operate accurately with gloves. Every interactive element in this UI is sized for gloved-finger use — toggle switches, mode buttons, and navigation elements are all generously sized with adequate touch targets.

🚨

Full-Screen System Failure — Cannot Be Missed

When a fault occurs, the entire screen turns red and shows the System Failure screen. This is non-dismissable without deliberate acknowledgement. In an industrial setting, an operator must not be able to accidentally swipe away or ignore a critical fault. The full-screen takeover ensures the fault gets the operator's complete attention.

⚙️

Auto vs Manual — Always Clearly Visible

Every AC unit row shows its operating mode badge alongside its ON/OFF status. An operator must never have to guess "is this running on automation or did someone manually switch it?" Confusing Auto and Manual modes in an industrial setting can lead to equipment running when it shouldn't — or not running when it must.

🔐

Login + Admin Permissions — Safety by Design

Not all operators should be able to change all settings. The login system and user management page ensure that only authorized personnel can access factory reset, recipe changes, or user management. Destructive actions (like factory default) also require a confirmation modal — preventing accidental critical operations.

📊

Trends Page — Data for Maintenance Teams

Industrial equipment failure is expensive. The Trends page shows historical performance data that allows facility managers to identify patterns — a unit that turns off every day at 2pm might have a thermal issue. This moves the system from reactive (fixing failures) to predictive (preventing them).

09 — Learnings & Reflections

What This Project Taught Me

Industrial UX Is a Completely Different Discipline

Consumer app design principles don't always apply to industrial HMI. Operator safety, glove-use, high-noise environments, and critical state visibility take complete precedence over aesthetics or delight.

Color Must Be Purely Functional

In industrial UI, color is a language — not a style choice. Using green for decoration or red for emphasis outside of status contexts would create real operational confusion and potentially endanger equipment.

Designing for Non-Digital Users

Factory technicians are experts in electrical systems — not software. Every interaction had to be self-explanatory without tutorials, tooltips, or onboarding flows. Clarity and convention matter above all.

Designing for Failure States First

In consumer apps, error states are afterthoughts. In industrial systems, the error state is the most critical screen. Designing the System Failure screen was the starting point — not the last thing added.

"

Industrial UI design taught me that the best interface is one that the user never has to think about — it just tells them exactly what they need to know, exactly when they need to know it. In critical systems, the design disappears and only the information remains.

— Design Reflection · AC Control Panel Project · Adroit Control Engineering
10 — Conclusion

Final Thoughts

This project represents one of the most technically demanding UI design challenges I've worked on — designing for an environment where clarity, error prevention, and instant comprehension are non-negotiable. The AC Control Panel HMI demonstrates that UI/UX design is not just about making things look beautiful — it's about making complex systems safe, efficient, and human-centered, even in the most demanding industrial environments.

Working on a real product for a real engineering company gave me a level of design rigor that no practice project can replicate. Every screen had to work — not just look good. This project is my most technically grounded piece of design work, and proof that UI/UX design plays a critical role even in traditional industrial engineering.
