01 — Overview

About Foodie

Foodie is a full-featured food delivery mobile application designed from scratch — covering the complete user journey from app launch to order delivery. The project includes a comprehensive component-based design system, multi-screen user flows for browsing, ordering, tracking, and account management, along with variant exploration for key UI components.

The Figma file contains 30+ screens organized across multiple user flows, with reusable components (cards, buttons, modals, bottom sheets, navigation bars) built as Figma components with variants. The design reflects real-world food delivery patterns inspired by Zomato, Swiggy, and Uber Eats — but with a distinct visual identity.

02 — Problem & Opportunity

What Problem Does Foodie Solve?

Food delivery apps are used daily by millions — yet many users still experience friction at every step. Discovery is overwhelming, menus are cluttered, checkout is lengthy, and order tracking feels disconnected. Foodie was designed to address these pain points with a fast, intuitive, and delightful ordering experience.

⚠ User Pain Points

  • Too many choices with no smart filtering — decision paralysis
  • Confusing menu layouts — hard to find popular or recommended items
  • Multi-step checkout flows with too many friction points
  • No real-time delivery tracking — users feel left in the dark
  • Account and reorder flows buried deep in navigation
  • Inconsistent design — each screen feels disconnected

✅ Foodie's Solution

  • Smart home feed with categories, featured & trending restaurants
  • Clean restaurant menu with image-led item cards and quick add
  • Streamlined cart → address → payment → confirm checkout flow
  • Live order tracking screen with delivery status updates
  • Profile with order history, saved addresses, and quick reorder
  • Consistent component system ensures visual harmony throughout

07 — Key UX Decisions

The Thinking Behind the Design

🎨

Red as the Primary Brand Color

Red is psychologically linked to appetite stimulation and urgency — both are desirable in a food delivery context. It creates a bold, memorable brand identity that stands out on the home screen alongside other apps. The red is used consistently for primary actions, reinforcing where users need to tap.

📸

Food Photography as the Primary UI Element

Real food images dominate restaurant cards, item cards, and the restaurant hero. Food buying decisions are heavily visual — a high-quality photo of a dish does more conversion work than any amount of copy. The UI is designed to frame these images, not compete with them.

⚡

Quick Add Button — Always Visible

Every food item card features a "+" button that adds to cart without navigating to the detail page. This dramatically speeds up the ordering flow for users who already know what they want. The detail page is an option, not a requirement.

📍

Location Always Visible in the Header

Delivery address is shown at the top of every main screen. This prevents confusion when users have multiple saved addresses and ensures they always know where their order will go before they start browsing — reducing checkout-time address-change frustration.

📊

Step Indicator in Checkout

A progress bar showing "Address → Payment → Review" reduces checkout abandonment by giving users a clear sense of how close they are to completing the order. Users are less likely to abandon when they can see they're 80% done.

🔁

Component System for Consistency & Scalability

Building restaurant cards, item cards, buttons, and bottom sheets as Figma components with variants means design changes propagate globally. It also makes the prototype more realistic and ensures a developer-friendly handoff with clear component definitions and states.

09 — Learnings & Reflections

What I Learned

Mobile Design Demands Thumb-First Thinking

Every important action in Foodie — search, add to cart, checkout, track — is reachable in the bottom third of the screen. Designing for thumb reach zones made the app feel effortless to use one-handed.

Components Are Worth the Upfront Investment

Building restaurant cards and item cards as Figma components early saved enormous time when changes needed to be applied across 30+ screens — proving the value of a component-first approach.

The "Empty State" Matters More Than You Think

Designing empty cart, no results, and offline states showed me that these "edge case" screens are often the moments that most define user trust in an app.

Flow Before Screens

Mapping all 6 user flows before designing a single screen prevented dead ends and forced me to think about every transition — resulting in a more cohesive product rather than a collection of beautiful but disconnected screens.
